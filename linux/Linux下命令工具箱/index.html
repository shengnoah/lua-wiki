Linux下命令工具箱

z.lua + ripgrep + fzf + fish shell + starship + exa + lazygit + byobu + spacevim + zed

z.lua目录跳转

zsh配置
.zshrc
eval "$(lua /path/to/z.lua --init zsh)"
fish配置
`~/.config/fish/conf.d/z.fish`
lua /path/to/z.lua --init fish | source

跳转到有hexo的目录
z -i hexo
https://github.com/skywind3000/z.lua

riggrep高级查找功能

查到当前目录的所有文件，含有关键字"rbenv"
rg -rn "rbenv"

fzf 模糊查找工具

跳转到检索目录
cd $(find * -type d | fzf)
跳转到检索文件
vim $(fzf)
zed $(fzf)
完整安装有按键快捷键绑定
sudo $(brew --prefix)/opt/fzf/install
Ctrl-t(bash)
Ctrl-r(zsh)
Atl-c(fish)

fish shell和好用的shell工具

https://fishshell.com/
conda init zsh

starship提示符配置工具

在Fish shell中启动starship
 ~/.config/fish/config.fish
starship init fish | source

https://starship.rs/

exa增加版本的ls

lazygit命令窗口的git工具

byobu高级化的tmux

这两个工具都属于生产力工具，平时使用的概率频繁，用好了日积月累可以节省很多时间，花点时间学习，可以节省很多不必要的时间花费。

摸索对比了byobu和 tmux ，确认是byobu生产性更高，并且支持 tab页，这种生产方式，本身就比tmux高校，而且快捷按键， tmux.是 control + b 命令， byobu是control + a 命令，并且还支持F区的功能快捷按键，比tmux的快捷按键更方便，一个功能有都快捷按键定义。

平时用的比较的终端是Warp和Tabby，因为每个工具对键盘的F功能区上的按键定义不一样，有的时候会出现冲突的，用control + a 命令的形工，与终端软件本身的冲突是最小的。

Tabby操作

shift + F2横切 [ ctrl + a | ]
ctrl + a % 竖切

shift + 上下左右， 移动屏幕。

ctrl + a 上下左右， 移动屏幕。

[F3]ctrl + a n 向前移动tab
[F4]ctrl + a p 向后移动tab

[F6]ctrl + a d 退出
tmux的退出是ctrl + b d

spacevim配置完各种常用插件neovim

ohmyz.sh

官网
https://ohmyz.sh/#install
主题（prompt提示符的风格不一样，路径和提示符分两行显示）
https://git.sjtu.edu.cn/sjtug/ohmyzsh/-/blob/master/themes/bira.zsh-theme

thefuck 自动纠正命令行错误

brew install thefuck

Zsh主题插件管理工具antigen

cd ~/.
curl -L git.io/antigen > .antigen.zsh

相关依赖

pip3 install pygments

autojump是Python写的， z.lua是用lua写的，直观速度上z.lua速度可以。
